About Manicule

This website serves as a demo of the Manicule application version 3 for exploring unique books and manuscripts. In this version, manuscript data is provided as a VisColl model in JSON format, as generated by the VCEditor tool.

Manicule can be used to either generate a standalone website, or referenced as a JavaScript library embedded in another website. It uses Web Components, an open standard that makes it possible to use custom HTML tags to add functionality.

Usage

Export a VisColl model file in JSON format and put it either in the data/ folder in this application, or somewhere accessible on the internet. In the example below, a model file was added as data/ms-codex-123/ms-codex-123.json. If the file is on a remote server, provide the full URL to the file on that server instead.

Import the Manicule JavaScript library as a JS module with: 

<script type="module" src="manicule/index.js"></script>

Then add the following HTML tag wherever you'd like to embed the document exploration: 

<collation-model 
  path="data/ms-codex-123/ms-codex-123.json">
</collation-model>

Inside the collation-model tag, add additional custom tags provided by Manicule to show spreads, individual leaves, thumbnails, folio structure, or a complete annotated tour. Use View Source in your browser to see how these examples are described.

Available tags

<spread-viewer>

Spreads are numbered incrementally from 0 to the last spread in the manuscript. By default, the spread viewer will start at the first spread and allow forward/backward navigation via <spread-nav> buttons.

<nav-strip>

Draws each page in linear order from the recto side of the first leaf to the verso side of the last leaf. Clicking on a page image in the nav-strip will jump any corresponding <spread-viewer> to that page.

<structure-view>

The structure view visualizes structural components of the manuscript, including which pages are conjoined, gathered, or inserted. If provided in the VisColl model, it will display information about material of the pages, whether pages are missing from the manuscript, and how they may be stitched together.

Examples

  • MS Codex 1077: contains an IIIF manifest, leaf terms, and some missing leaves
  • MS Codex 107: contains stitching information, no IIIF manifest or page images
